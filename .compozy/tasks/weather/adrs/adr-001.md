# ADR-001: Key-less services (Open-Meteo + ip-api.com)

## Status

Accepted

## Context

The CLI needs current weather and approximate geolocation. Most weather and geo
providers require an API key, which forces account setup, secret handling, and
makes the project non-reproducible for anyone cloning it.

## Decision

Use **Open-Meteo** (`api.open-meteo.com`) for current temperature and
**ip-api.com** for IP geolocation. Both are free and require no API key.

## Consequences

- Anyone can build and run the CLI, and re-run the test fixtures, with no secrets.
- Rate limits apply on the free tiers; acceptable for a single-user CLI.
- Both clients take an injectable base URL so tests never touch the real network.
